# 开发规范

## 代码风格

- 使用 ESLint + Prettier 自动格式化
- 单引号、分号、120 字符行宽
- 保存时自动格式化（推荐安装编辑器插件）

## Git 提交规范

使用 `pnpm run commit` 提交代码，自动触发：
1. ESLint 修复
2. Prettier 格式化
3. 提交信息验证

### 提交类型

- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档变更
- `style`: 代码格式
- `refactor`: 重构
- `perf`: 性能优化
- `test`: 添加测试
- `chore`: 构建过程或辅助工具的变动

## 最佳实践

1. **请求取消**：页面卸载时记得取消请求
2. **环境切换**：使用 `switchEnv()` 切换环境
3. **Token 管理**：根据需要配置自动刷新
4. **页面缓存**：复杂参数使用缓存传递
